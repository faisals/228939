7-Day Solana On-Chain Data Curriculum (2024/2025)

This curriculum is designed for an intensive 7-day, 8-hour-per-day learning experience. The goal is to become proficient in programmatic interaction with the Solana blockchain, including writing and deploying on-chain programs using the Anchor framework. Each day includes key topics, hands-on exercises, and practical insights for mastering on-chain data.

Day 1: Fundamentals of Solana & Environment Setup

Learning Goals

  • Understand the basic concepts of Solana (Proof of History, architecture, validators, cluster types).
  • Install and configure development tools (Rust, Solana CLI, Anchor CLI).
  • Familiarize yourself with key Solana CLI commands.
  • Write a simple script to interact with the Solana blockchain (fetch wallet balance).

Topics to Cover (Approx. 8 hours)

  1. Solana Basics (2 hours)
    • How Solana processes transactions (Proof of History & Tower BFT overview).
    • Cluster types: Devnet, Testnet, Mainnet-beta, and Localnet.
    • Key pairs, wallets, and addresses.
  2. Development Environment Setup (1 hour)
    • Install Rust & set up the Rust toolchain (Nightly may be needed for Anchor).
    • Install the Solana CLI tools (solana --version).
    • Install Anchor CLI (anchor --version).
  3. Solana CLI Crash Course (2 hours)
    • Generating keypairs (solana-keygen new).
    • Requesting airdrops on devnet (solana airdrop).
    • Checking balances and transaction history.
    • Transferring SOL between accounts.
  4. Intro to Programmatic Interaction (2 hours)
    • Basic example in Python or TypeScript using @solana/web3.js or solana-py.
    • Connecting to devnet and fetching wallet balance programmatically.
  5. Wrap-up & Q&A (1 hour)

Exercise (Day 1)

Goal: Write a short script to fetch and display your wallet’s balance programmatically.

  • Create or import a keypair for devnet.
  • Airdrop some SOL to your devnet wallet (solana airdrop).
  • In Python or TypeScript, connect to devnet and retrieve your wallet’s balance.
  • Print the balance to the console.

Day 2: Solana Transaction Model & Reading On-Chain Data

Learning Goals

  • Deeply understand Solana transactions and instructions.
  • Learn how to decode transaction data.
  • Explore how on-chain accounts store and structure data.

Topics to Cover (Approx. 8 hours)

  1. Transaction & Instruction Model (2 hours)
    • Anatomy of a transaction: signatures, message, instructions, recent blockhash.
    • Role of signers vs. read-only accounts in transactions.
    • Fee structure on Solana.
  2. Reading Transaction History (2 hours)
    • Using the Solana CLI to fetch transaction details.
    • Using @solana/web3.js or solana-py to query recent transactions.
    • Decoding transaction logs and events.
  3. Accounts & Data Storage (2 hours)
    • Solana accounts model: rent, data size, ownership.
    • Program Derived Addresses (PDAs) introduction (briefly – deeper dive on Day 4).
    • Common pitfalls in storing data on-chain (rent-exempt rules, etc.).
  4. Practical Examples (1 hour)
    • Walk through real transactions in the Solana Explorer.
    • Decode instructions from a popular protocol (e.g., a DEX transaction).
  5. Wrap-up & Q&A (1 hour)

Exercise (Day 2)

Goal: Create a script that fetches and decodes transaction data for a specific address.

  • Pick a public program or your own devnet address with recent transactions.
  • Fetch the transaction signatures for the last N transactions.
  • Decode each transaction to show signatures, involved accounts, logs or events.
  • Print results to the console in a readable format.

Day 3: Introduction to Writing Solana Programs (Anchor Framework)

Learning Goals

  • Understand the structure of a Solana program (smart contract) in Rust.
  • Set up and use the Anchor framework for streamlined development.
  • Create, build, and deploy your first Solana program to devnet.

Topics to Cover (Approx. 8 hours)

  1. Why Anchor? (1 hour)
    • Overview of Anchor’s features: #[derive(Accounts)], macros, PDAs, error handling.
    • Comparison to raw Solana program development in Rust.
  2. Project Structure & Anchor Basics (2 hours)
    • Anchor CLI commands (anchor init, anchor build, anchor deploy).
    • Anatomy of an Anchor program: lib.rs, Cargo.toml, IDL generation.
  3. Writing a Simple “Hello World” Program (2 hours)
    • Define a context struct and handler function.
    • Understand how accounts are passed into the program.
    • Build and deploy to devnet.
  4. Interacting with the Program (2 hours)
    • Using Anchor tests or a simple client script to send transactions.
    • Verifying logs in the Solana Explorer or using solana logs.
  5. Wrap-up & Q&A (1 hour)

Exercise (Day 3)

Goal: Build and deploy a simple “Hello World” Anchor program.

  • Create a new Anchor project with anchor init hello_world.
  • Modify the program to store a simple string or counter in an account.
  • Deploy the program to devnet.
  • Write a test (JavaScript/TypeScript or Rust) that calls the program’s method to update on-chain data (e.g., increment a counter).
  • Confirm the state changes on-chain by checking the account data with your client script.

Day 4: Advanced Data Handling: PDAs, Seeds, and Complex Data Structures

Learning Goals

  • Master the use of Program Derived Addresses (PDAs) and seeds.
  • Learn how to design and store complex data structures in accounts.
  • Understand account constraints, security checks, and best practices.

Topics to Cover (Approx. 8 hours)

  1. Deeper Dive into PDAs (2 hours)
    • Why PDAs are secure (Ed25519 point off the curve).
    • Seeds and bumps.
    • Common patterns for generating PDAs (e.g., for user profiles, “vault” accounts).
  2. Complex Data Serialization (2 hours)
    • Using Anchor’s #[account] macro to define data schemas.
    • Handling vectors, maps, or advanced data structures.
    • Memory and rent considerations for large accounts.
  3. Program Constraints & Security (2 hours)
    • Anchor’s constraint system (e.g., #[account(mut, has_one = authority)]).
    • Preventing data tampering and ensuring only valid signers can modify data.
  4. Hands-On Examples (1 hour)
    • Creating multiple PDAs for different user states or items in a game/market.
  5. Wrap-up & Q&A (1 hour)

Exercise (Day 4)

Goal: Create a program that uses PDAs to manage multiple user data accounts.

  • Define a UserProfile structure (e.g., username, reputation, etc.).
  • Use a PDA to derive a unique address for each user’s profile.
  • Write instructions to create and update these user profiles.
  • From a client script, create a profile and then update it with mock data.
  • Confirm each user’s data is stored in a separate PDA and verify the on-chain data.

Day 5: Indexing and Off-Chain Data Pipelines

Learning Goals

  • Understand how to index on-chain events and account data for scalable dApps.
  • Explore existing indexing solutions (Helius, LogSnag, or custom event listeners).
  • Build a small indexing service to watch a program’s data changes.

Topics to Cover (Approx. 8 hours)

  1. Introduction to Indexing (2 hours)
    • Why indexing is necessary (real-time dashboards, analytics, etc.).
    • Event vs. polling approaches.
    • Webhooks, WebSockets, or RPC-based polling.
  2. Existing Tools & Services (1 hour)
    • Helius, LogSnag, or other third-party indexing solutions.
    • Building a custom indexer using a local database (e.g., PostgreSQL).
  3. Implementing a Custom Indexer (3 hours)
    • Listening for program events/logs.
    • Decoding events using your program’s IDL (Anchor).
    • Storing relevant data in an off-chain database.
  4. Hands-On with an Indexer (1 hour)
    • Connect a Node.js/Python script that fetches logs/events for your Day 4 program.
    • Show real-time updates whenever a user updates their profile.
  5. Wrap-up & Q&A (1 hour)

Exercise (Day 5)

Goal: Build a basic indexer service that listens for user profile changes in your Day 4 program.

  • Create a Node.js or Python service that checks for new transactions involving your program’s ID (polling or WebSocket).
  • Decode relevant logs or events (e.g., “ProfileUpdated”) and store them in a local/cloud DB.
  • Create a simple dashboard (command-line or web-based) to display the recorded profiles and changes in real time.

Day 6: Debugging, Testing, and Performance

Learning Goals

  • Learn best practices for debugging Solana programs and transactions.
  • Understand how to write comprehensive tests for your Solana programs.
  • Explore performance considerations and optimization techniques.

Topics to Cover (Approx. 8 hours)

  1. Debugging Techniques (2 hours)
    • Using solana logs and Anchor’s simulate feature.
    • Common runtime errors and how to fix them (e.g., AccountNotEnoughKeys, InstructionError).
    • Reading program logs in the Explorer vs. local environment.
  2. Testing Strategies (2 hours)
    • Anchor’s built-in testing environment (anchor test).
    • Integrating with Mocha/Jest (for TypeScript-based tests).
    • Local test scenarios with solana-test-validator.
  3. Performance & Concurrency (2 hours)
    • Solana’s parallel execution model (accounts-based concurrency).
    • Minimizing account data size and transaction overhead.
    • Rent optimization (closing unused accounts).
  4. Hands-On Debugging & Optimization (2 hours)
    • Intentionally break a transaction and diagnose the error.
    • Optimize user profile program from Day 4 to reduce transaction/account size.

Exercise (Day 6)

Goal: Debug a failing transaction in your program and then improve performance.

  • Modify your Day 4 program so an instruction fails intentionally (e.g., incorrect account constraints).
  • Run a transaction that triggers the failure; capture logs and error messages.
  • Fix the bug, confirm the transaction now succeeds.
  • Try to reduce account/data footprint to optimize rent costs. Document changes.

Day 7: Final Project & Expert-Level Topics

Learning Goals

  • Integrate everything learned into a full dApp workflow.
  • Explore advanced Solana features (CPI calls, cross-program interactions, zero-copy deserialization).
  • Complete an end-to-end project demonstrating mastery of on-chain data handling.

Topics to Cover (Approx. 8 hours)

  1. Cross-Program Invocations (CPI) & Composability (2 hours)
    • Calling into other programs (e.g., token program, associated token program).
    • Using CPIs to extend your program’s functionality.
  2. Zero-Copy Deserialization & Other Advanced Techniques (1 hour)
    • Reducing serialization overhead in performance-critical apps.
    • Creating slices of account data for partial updates.
  3. Security & Auditing (1 hour)
    • Common Solana security pitfalls.
    • Tools and best practices for auditing code.
  4. Final Project Overview (2 hours)
    • Brainstorm a real-world application to build or extend in ~1 day.
    • Outline a minimal end-to-end solution: on-chain program + indexer + front-end or CLI.
    • Implementation plan (account structure, instructions, off-chain data pipeline).
  5. Project Work (1-2 hours)
    • Begin coding your final project or setting up the skeleton.
  6. Wrap-up & Q&A (1 hour)

Exercise (Day 7) – Final Project

Goal: Combine all learnings into one final dApp or advanced script.

  • Example Project: “Distributed Marketplace”
    • On-chain: A program that lets users list items (data structures) with metadata and a price.
    • PDAs: Each item is stored in a PDA.
    • CPI: Optional use of the SPL Token program to accept custom token payments.
    • Off-chain Indexer: Watch for new item listings and store them for fast querying.
    • Front-end/CLI: Minimal interface to display and “purchase” items.
  • Adjust scope or choose a different project (the key is demonstrating the full flow).
